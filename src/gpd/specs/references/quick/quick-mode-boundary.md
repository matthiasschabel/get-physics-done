<quick_mode_boundary>
Quick mode is for small, ad-hoc tasks in an initialized project: local calculations, dimensional checks, order-of-magnitude estimates, one-file numerical spot-checks, small plot generation, and targeted bibliography or formula lookups.

Default quick authoring is a small-task path. It keeps the project contract gate visible but does not load full reference artifacts, literature maps, proof-review status, or publication runtime context unless the task explicitly needs that context.

Select the `reference_context` staged init only when the answer depends on active project anchors, existing reference artifacts, literature/research-map files, or a specific source lookup. Do not select it for pure local algebra, dimensional analysis, unit conversion, simple numerical evaluation, formatting, or a one-file sanity check.

Promote out of quick when the task becomes multi-step planned work, needs traceability to requirements, requires convergence or parameter-sweep validation, or might produce paper-cited evidence. Use `gpd:add-phase`, `gpd:insert-phase`, `gpd:plan-phase <phase>`, or the dedicated verification/publication command instead.
</quick_mode_boundary>
